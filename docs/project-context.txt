2) ARCHITECTURE
================================================================================

2.1 High-level service map
- Browser (React SPA) and Flutter clients talk to the same Express HTTP API under /api/*.
- Express uses Prisma -> primary PostgreSQL (system of record).
- Optional Redis (cache/session style usage via lib/cache.ts) — server warns if down.
- Optional NATS (lib/bus.ts) for publish-only domain events — server warns if down; consumers not required for app to run.
- Secondary PostgreSQL (media_db) + MinIO for explorer/media pipeline (lib/mediaDb.ts, routes/media.ts).
- Traefik (when compose stack fully up) routes path prefixes to Minio, Metabase, Portainer, Dozzle, Flutter static, and web-app.

2.2 API gateway
- Traefik acts as reverse proxy when using full Docker stack on port 80.
- There is no separate API Gateway product; the Node server is the API edge for /api.

2.3 Databases
- PostgreSQL 16 (primary): users, orders, KYC, contracts, etc.
- PostgreSQL 16 (media): metadata for media feed / explorer.
- Not MongoDB.

2.4 Authentication
- JWT access + refresh (routes/auth.ts). Cookie/header patterns per implementation in auth middleware.
- Google OAuth endpoints present.
- WebAuthn (@simplewebauthn) registration and login options under /api/auth.
- Protected routes use authenticate / requireRole from lib/auth.middleware.ts (imports in routes use .js extension after compile).

2.5 Message queue
- NATS is used as a lightweight message bus (publish); not RabbitMQ.
- Redis is a cache client (ioredis), not a durable task queue in the documented sense.

- Dart SDK constraint in pubspec: >=3.0.0 <4.0.0; local tool version file may show e.g. 3.41.x (Flutter/Dart toolchain on dev machine).

- React ^19, Vite ^6, react-router-dom ^7.
- Styling: Tailwind CSS v4 (@tailwindcss/vite), tailwind-merge, clsx; motion (Framer successor) for animation; lucide-react icons.

5) PRODUCT AND BUSINESS LOGIC
================================================================================

5.1 One-liner
Neighborly is a local services marketplace: customers request work, providers fulfill via companies/workspaces, with orders, matching, contracts, chat, and payments on platform.

5.2 User types (Prisma UserRole and product language)
owner, platform_admin, developer, support, finance, customer, provider, staff (plus StaffRole for internal staff flavors). KYC rules: personal KYC before provider; business KYC before core provider business features (see docs/AGENTS.md).

5.3 Major implemented features (see docs/ROADMAP.md phases F0–F8 evidence)
- Hierarchical categories and service catalog with dynamic field schemas (F0).
- Admin CRM users module partial (F1 in progress).
- Admin KYC queues and form builder (F2); customer/provider KYC v2 flows (F3).
- Flutter shell navigation rework (F4).
- Order wizard MVP, admin orders (F5); Flutter order wizard explicitly deferred in roadmap.
- Matching engine MVP auto vs round-robin pool (F6).
- Order-scoped chat, PII moderation, optional translation layer (F7).
- Versioned order contracts, AI-assisted draft endpoints, payment gate tied to approved contract, minimal payment ledger (F8 partially complete; templates/disputes/settlement depth deferred).

5.4 Roadmap / in-flight (from docs/ROADMAP.md)
- F1 admin users UX completion.
- F8 remaining: templates, disputes, settlement automation; continued payment/contract depth.
- F9 lost-deal analytics and provider scorecards — deferred post-Flutter parity.
- F10 weekly bulletin and market insights — deferred post-Flutter parity.
- Cross-cutting: Flutter parity for F5–F8, i18n/RTL polish, observability depth.

5.5 AI-related features
- Google Gemini client (@google/genai) in web bundle for consultant / BI style features (see src/services/geminiService.ts, aiService.ts, BusinessIntelligence.tsx per docs/ARCHITECTURE.md).
- Order wizard AI coach (referenced in roadmap evidence).
- Contract draft-from-AI endpoint POST /api/orders/:orderId/contracts/draft-from-ai.
- Flutter: google_generative_ai and AI consultant screen.
- Matching is rules/engine code under lib/matching/, not an external ML model.
