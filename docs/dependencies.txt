--------------------------------------------------------------------------------
3a) Cryptography, hashing, tokens, and authentication (package.json + runtime)
--------------------------------------------------------------------------------
These are not a custom in-app crypto model; they are standard libraries / primitives:

- bcrypt — password hashing (not general-purpose symmetric/asymmetric encryption).
- jsonwebtoken — JWT sign/verify (typically HMAC or RSA; transitive deps like jwa / buffer-equal-constant-time in the npm tree).
- @simplewebauthn/browser / @simplewebauthn/server / @simplewebauthn/types — WebAuthn (FIDO2) on COSE/CBOR and browser/server crypto APIs.
- google-auth-library — Google OAuth token verification (uses Node crypto).
- helmet — HTTP security headers (does not replace TLS).
- Node built-in module: crypto (no standalone package.json entry).
- Docker image: OpenSSL on node:20-alpine for Prisma / DB TLS clients.

Flutter (current pubspec): no dedicated crypto package (e.g. pointycastle) declared; transport security is HTTPS/TLS to server/proxy; shared_preferences stores plain strings (use Keychain/Keystore for sensitive mobile tokens when needed — out of scope for this file).

@google/genai — Google generative AI client; not direct application payload encryption.
